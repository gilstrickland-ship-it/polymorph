# Report R6 — Production-readiness checklist

The integration test against `@primer/primitives` validates the **runtime + adapter +
loader** surfaces against a real published design system. The **publishing /
distribution** surfaces — what an FI needs to actually install and use Polymorph in
production — are covered here.

---

| Item | Status |
|---|---|

## ⚠️ Real-world findings that the contract DOESN'T currently address

These came out of the lint report (R2) and aren't bugs — they're honest gaps the contract
could grow into:

| Finding | Future spec? |
|---|---|
| `CONTRAST_ON_ACTION_LOW` doesn't know about AA Large (3:1) escape for button text | Possible: split text-on-action by font-size tier |
| `BORDER_DEFAULT_LOW` treats decorative borders as informational | Possible: a per-token "decorative" flag in the manifest |
| Reduced-motion: native runtimes still need a host-supplied "is reduced motion on?" query | The transform exists; the platform query doesn't (out of contract scope) |

These are captured here as known gaps; future spec cycles would refine them.
