# Polymorph docs

Home for the **standard spec site** and **adoption guides**.

- **The standard** — the DTCG-extended contract, the semantic vocabulary reference, component-token and theme-mode conventions, and versioning rules (authored alongside `@polymorph/spec`).
- **Vendor adoption guide** — how an SDK author codes against the contract (semantics, slots, optional component mapping) so one build adapts to N design systems.
- **FI adoption guide** — how a financial institution authors and delivers its theme (token authoring, the three loaders, advisory a11y lint).

> Populated as the specs land, starting with **Spec A — The Contract**.
